## 1. ภาพรวมสถาปัตยกรรม (Architecture)

```
┌────────────────────────────────────────────────────────────┐
│                      Client (Browser)                       │
│   React + Vite + TailwindCSS + React Router + Recharts     │
└──────────────────────────┬──────────────────────────────────┘
                           │ HTTPS / REST API (JSON + JWT)
┌──────────────────────────▼──────────────────────────────────┐
│                      Backend API Server                     │
│   Node.js + Express + Prisma ORM + JWT Auth + Zod Validate │
│   - Auth / Role Middleware                                  │
│   - Business Logic (Invoice, Billing, Report)               │
│   - Cron Jobs (แจ้งเตือนค้างชำระ, ปิดงวดรายเดือน)           │
└──────────────────────────┬──────────────────────────────────┘
                           │
┌──────────────────────────▼──────────────────────────────────┐
│                  Database: PostgreSQL (Prod)                │
│                  / SQLite (Dev, Single-tenant)              │
└─────────────────────────────────────────────────────────────┘
                           │
                    ┌──────┴──────┐
                    │   Storage   │  (ใบเสร็จ/รูปถ่าย — Local FS หรือ S3)
                    └─────────────┘
```

**Stack สรุป**
- Frontend: React 18 + Vite + TypeScript + TailwindCSS + shadcn/ui + Recharts + React Hook Form + Zod
- Backend: Node.js 20 + Express + Prisma + SQLite/PostgreSQL + JWT + bcrypt + node-cron
- Infra: Docker Compose (dev/prod) — รันบนเครื่อง/VPS/LAN ได้

## 2. ระบบ Role & Permission

| Role | สิทธิ์ |
|------|-------|
| **Admin** | ทุกสิทธิ์ — จัดการผู้ใช้, ตั้งค่าระบบ, อนุมัติรายจ่าย, ดูรายงานทั้งหมด |
| **Manager** | จัดการล็อค, ผู้เช่า, ออกใบแจ้งหนี้, บันทึกรายรับ-รายจ่าย, ดูรายงาน |
| **Staff** | บันทึกรายรับ (เก็บเงิน), จดมิเตอร์น้ำ, บันทึกรายจ่ายทั่วไป |
| **Accountant** | ดูรายงานการเงินทั้งหมด, Export งบ, ไม่แก้ไขธุรกรรม |
| **Viewer** | ดู Dashboard อย่างเดียว (เช่น เจ้าของตลาด) |

## 3. Data Model (ฐานข้อมูล)

### 3.1 User & Auth
```
User(id, username, passwordHash, fullName, role, email, phone, isActive, createdAt)

LockContract(id, lockId, tenantId, startDate, endDate, rentAmount, commonFee, depositAmount, status[ACTIVE|ENDED|TERMINATED])

### 3.3 รายรับ
```
Invoice(id, invoiceNo, contractId, type[RENT|COMMON|WATER|ELECTRIC|OTHER], period(YYYY-MM), dueDate, amount, status[PENDING|PAID|OVERDUE|PARTIAL], issuedAt, paidAt)
InvoiceItem(id, invoiceId, description, quantity, unitPrice, amount)
Payment(id, invoiceId, amount, method[CASH|TRANSFER|QR], paidAt, receivedByUserId, receiptNo, note)
WaterMeterReading(id, lockId, readingDate, meterStart, meterEnd, unitsUsed, ratePerUnit, totalAmount, invoiceId)
ElectricMeterReading(id, lockId, readingDate, meterStart, meterEnd, unitsUsed, ratePerUnit, totalAmount, invoiceId)
OtherIncome(id, date, category, description, amount, receivedByUserId, note)
```

### 3.4 รายจ่าย
```
ExpenseCategory(id, code, name, type[OPERATION|CONSTRUCTION|EVENT|UTILITY|GENERAL])
Expense(id, date, categoryId, description, amount, approvedByUserId, receiptNo, attachmentUrl, note)

ConstructionProject(id, name, contractor, budget, startDate, endDate, progressPct, status)
ConstructionExpense(id, projectId, date, description, amount, receiptNo)

Event(id, name, eventDate, budget, description)
EventExpense(id, eventId, subCategory, description, amount)
EventIncome(id, eventId, description, amount)

UtilityBill(id, period(YYYY-MM), electricAmount, waterAmount, internetAmount, garbageAmount, wasteWaterAmount, totalAmount, paidAt)
```

### 3.5 ทรัพย์สิน
```
AssetCategory(id, name[LAND|BUILDING|UTILITY_SYSTEM|EQUIPMENT])
Asset(id, code, categoryId, name, description, acquireDate, costPrice, usefulLifeYears, depreciationMethod, accumulatedDepreciation, currentValue, location, status)
AssetDepreciationLog(id, assetId, period, amount, bookValue)
```

### 3.6 รายงาน (Views / Cache)
```

## 4. API Endpoints (REST)

### Auth
- `POST /api/auth/login` → JWT
- `POST /api/auth/logout`
- `GET  /api/auth/me`

### Users (Admin)
- `GET|POST|PATCH|DELETE /api/users`

### Locks & Tenants
- `GET|POST /api/zones`
- `GET|POST|PATCH /api/locks` — รองรับ filter `?status=VACANT&zoneId=`
- `GET|POST|PATCH /api/tenants`
- `GET|POST|PATCH /api/contracts` — ออกสัญญา, ต่อสัญญา, ยกเลิก

### Billing & Income
- `POST /api/invoices/generate-monthly` — สร้างใบแจ้งหนี้ค่าเช่า+ส่วนกลางทั้งตลาดสำหรับเดือนที่ระบุ
- `GET|POST /api/invoices`
- `POST /api/invoices/:id/payments` — บันทึกการชำระ
- `POST /api/water-readings` — จดมิเตอร์น้ำ (auto-สร้าง invoice ค่าน้ำ)
- `POST /api/electric-readings` — จดมิเตอร์ไฟ (auto-สร้าง invoice ค่าไฟ)
- `GET|POST /api/other-income`

### Expenses
- `GET|POST /api/expenses` — รายจ่ายภาพรวม + ทั่วไป
- `GET|POST /api/construction/projects` และ `/:id/expenses`
- `GET|POST /api/events` และ `/:id/expenses` `/:id/income`
- `GET|POST /api/utility-bills`

### Assets
- `GET|POST|PATCH /api/assets`
- `POST /api/assets/depreciate` — คำนวณค่าเสื่อมงวดปัจจุบัน

### Reports
- `GET /api/reports/daily?date=YYYY-MM-DD`
- `GET /api/reports/monthly?period=YYYY-MM`
- `GET /api/reports/dashboard` — KPI รวม
- `GET /api/reports/overdue` — ลูกค้าค้างชำระ
- `GET /api/reports/occupancy` — อัตราการเช่า
- `GET /api/reports/export?type=monthly&format=xlsx|pdf`

## 6. ฟีเจอร์พิเศษ (Automation)

| ฟีเจอร์ | รายละเอียด | เทคนิค |
|--------|----------|-------|
| สร้างใบแจ้งหนี้รายเดือนอัตโนมัติ | ทุกวันที่ 1 ของเดือน สร้าง Invoice ค่าเช่า+ส่วนกลางให้ทุก Contract ที่ ACTIVE | `node-cron` |
| แจ้งเตือนค้างชำระ | ทุกวันเช็ค Invoice เกิน dueDate → เปลี่ยนสถานะ OVERDUE + notification | `node-cron` |
| คำนวณค่าเสื่อมทรัพย์สินรายเดือน | Straight-line method ตามอายุใช้งาน | Cron สิ้นเดือน |
| ส่งใบแจ้งหนี้ทาง LINE/SMS | ผ่าน LINE Notify / Messaging API | Optional v2 |
| Dashboard Real-time | กราฟ 12 เดือน, Pie รายจ่าย, อัตราการเช่า | Recharts |
| Export | Excel (exceljs) / PDF (pdfkit) | Server-side |
| Audit Log | ทุก mutation บันทึก user + before/after | Prisma middleware |
| Backup | SQLite dump รายวัน หรือ pg_dump | Cron + S3 |

## 9. Security & Best Practice

- Password: bcrypt (cost 12)
- JWT: access token (15min) + refresh token (7d) เก็บใน httpOnly cookie
- Rate limit: `/auth/login` 5 req/min
- Input validation: Zod ทุก endpoint
- SQL Injection: ใช้ Prisma (parameterized)
- XSS: React escape + CSP header
- HTTPS only (prod)
- Role-based + Row-level ตรวจใน middleware
- Audit log ทุก mutation
- Backup DB รายวัน
